我们考虑一个多蜂窝网络场景，其中有 $N$ 个移动用户随机分布在由 $M$ 个基站覆盖的区域中。每个用户有且仅有一个独立且不可分割的计算任务。第 $i$ 个用户的计算任务定义为 $\mathbf{u}_i$，该任务可用一个五元组表示：

$\mathbf{u}_i = {D_i^{in}, D_i^{out}, \omega_i, s_i, \tau_i}$

其中：

$D_i^{in}$：任务的输入数据大小（单位：比特）

$D_i^{out}$：任务的计算结果数据大小（单位：比特）

$\omega_i$：完成任务所需的CPU周期数

$s_i$：任务类型，从 $S$ 种任务类型中选择

$\tau_i$：处理任务的最大允许时延（单位：秒）

基站集合表示为 $\mathbb{B} = {b_1, b_2, ..., b_M}$，每个基站 $b_j$ 都配备一个MEC服务器，用于处理移动用户的计算任务。基站 $b_j$ 的MEC服务器分配给用户 $i$ 的计算资源记为 $f_{ij}$。系统中配置了一个MEC控制器，通过控制信道收集用户的任务信息、移动性信息、各基站MEC服务器的缓存信息以及剩余计算资源，并基于这些信息集中做出卸载和迁移决策。
